# Bot Dashboard

Мини-дашборд для управления Yandex-ботами. Запускается независимо от основного дашборда.

## Что умеет

- **Боты** — запуск/остановка/логи, просмотр состояния (warmup_days, последний запуск, кол-во запросов)
- **Прокси** — добавить/заменить/удалить, статус бана
- **Google Docs** — просмотр и редактирование ссылок на документы с запросами
- **Автопилот** — оркестратор: автозапуск, авто-переключение warmup→target, суточное окно, задержка перезапуска

## Требования

- Node.js >= 18
- pnpm (или npm/yarn)
- Python 3 + yandex_bot установлены на машине

## Установка

```bash
cd bot-dashboard
cp .env.example .env
# Отредактировать .env: указать BOT_DIR
nano .env

pnpm install   # или: npm install
```

## Запуск (dev)

```bash
pnpm dev       # или: npm run dev
```

Открыть: http://localhost:4000

## Конфигурация (.env)

```env
PORT=4000

# Путь к папке yandex_bot (обязательно)
BOT_DIR=/home/user/yandex_bot
```

## Структура файлов yandex_bot

Дашборд читает и пишет следующие файлы внутри `BOT_DIR`:

| Файл | Назначение |
|------|-----------|
| `proxies.txt` | Список прокси |
| `outputs/proxy_cache.json` | Кэш проверенных прокси |
| `outputs/proxy_blacklist.json` | Забаненные прокси (капча) |
| `outputs/bot_states/bot_N_state.json` | Состояние каждого бота |
| `outputs/orchestrator.json` | Конфиг автопилота |
| `outputs/google_docs.json` | URL Google-документов |
| `logs/bot_N.log` | Логи каждого бота |

## Автопилот

В табе **Автопилот**:
1. Добавь боты (ID + сайт)
2. Настрой параметры: макс. параллельных, задержка перезапуска, суточное окно
3. Включи тумблер и нажми **Сохранить конфиг**

Режим (warmup / target) определяется автоматически: `warmup_days < 14` → warmup, иначе target.

Тик каждые 30 секунд. Конфиг перечитывается при каждом тике — изменения применяются без перезапуска.
