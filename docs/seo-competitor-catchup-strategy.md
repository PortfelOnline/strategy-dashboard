## Алгоритм улучшения (постепенно)

### Шаг 1 — После каждого батча проверять позиции
```
npm exec tsx scripts/analyze_top3.ts
```
- Для каждой улучшенной статьи запускать Яндекс SERP (100 результатов)
- Фиксировать вход в топ-100 / топ-50 / топ-20 / топ-3

### Шаг 2 — Детальный анализ конкурентов для статей в топ-50
Когда статья входит в топ-50 Яндекса:
```
npm exec tsx scripts/compare_competitors.ts
```
Что сравниваем:
- Количество слов (мы vs топ-3 конкурентов)
- Количество H2/H3
- Наличие FAQ (details/schema)
- Наличие таблиц
- Первый абзац — точность ответа на запрос
- Наличие интерактивного инструмента на странице

### Шаг 3 — Точечное улучшение
Если конкурент превосходит нас по какому-то параметру:

| Что не хватает | Действие |
|----------------|----------|
| Слов меньше конкурента | Enhance pass через `rewriteArticle` |
| Нет ответа на запрос в первых 100 словах | Переписать intro с answer-first |
| Меньше FAQ | Добавить FAQ-блок через enhance |
| Нет таблицы сравнения | Добавить таблицу через enhance |
| Слабый заголовок H1 | Переписать H1 под точный ключ |

### Шаг 4 — Публикация и индексация
После улучшения:
- IndexNow отправляет сигнал Яндексу и Google автоматически
- Ждём 3-14 дней для переиндексации
- Повторяем измерение позиций

## Особенности для Яндекса vs Google

### Google (уже работает)
- Рендерит JS → видит интерактивную карту
- Ценит длинный контент + FAQ-схемы + internal linking

### Яндекс (в работе)
- Слабо рендерит JS → карта в `<div id="map">` выглядит как пустой блок
- Ценит точное вхождение запроса в первые 100 слов
- Ценит domain authority (TIC) — строится медленно через ссылки
- Для навигационных/инструментальных запросов предпочитает страницы-инструменты

### Специфика карточных запросов ("найти участок со спутника")
Яндекс-топ — это инструменты (map.ru, kadastrmapp.online) с 300-400 словами.
Наша страница имеет карту через JS + 4000+ слов статьи.
Возможное улучшение: добавить статичный скриншот карты со спутника как `<img>` в начало статьи — Яндекс-бот увидит его без JS.
